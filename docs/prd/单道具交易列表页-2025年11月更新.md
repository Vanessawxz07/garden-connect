# 单道具交易列表页 - 2025年11月更新

> 本文档记录单道具交易列表页的现网版本需求，作为后续新游戏拓展专项的基线参考。

---

## 一、页面概述

单道具交易列表页展示某个特定道具相关的所有交易订单，是用户查找特定道具交易的核心页面。

---

## 二、页面结构

### 2.1 道具信息头部

**道具基础信息：**
- 道具大图
- 道具名称（H1）
- 道具分类标签
- 道具Value值
- 道具描述

**统计信息：**
- 当前活跃交易数
- 历史成交数
- Value趋势（如有）

**操作区域：**
- 创建交易按钮（包含此道具）
- 收藏道具按钮
- 分享按钮

### 2.2 交易方向Tab

**Tab选项：**
- All Trades（全部）
- Selling（出售 - Have此道具）
- Buying（求购 - Want此道具）

**交互逻辑：**
- 切换Tab刷新列表
- URL同步更新
- 保持其他筛选条件

### 2.3 筛选和排序

**筛选条件：**
- Value范围筛选
- 在线状态筛选（仅显示在线用户）
- 排除NLF筛选（排除我有的道具在NLF中的订单）

**排序选项：**
- 最新发布
- Value最高
- Value最低
- 最近活跃

**筛选器布局：**
- 桌面端：横向展开
- 移动端：收起为筛选抽屉

### 2.4 交易列表

**列表布局：**
- 卡片列表（每行一个订单）
- 响应式适配

**订单卡片内容：**

| 区域 | 内容 |
|------|------|
| 用户信息 | 头像、用户名、在线状态、信誉评分 |
| Have区域 | 道具列表、总Value |
| Want区域 | 道具列表/Accept Any、总Value |
| NLF提示 | 若有NLF则显示图标提示 |
| 操作区域 | 查看详情、快速报价按钮 |
| 时间信息 | 发布时间、最近活跃时间 |

**卡片交互：**
- 点击卡片跳转订单详情页
- 快速报价按钮打开报价弹窗
- Hover高亮效果

### 2.5 分页

**分页方式：**
- 底部分页器
- 显示总页数和当前页
- 支持跳转指定页

**每页数量：**
- 默认20条
- 可选：10/20/50条

---

## 三、功能说明

### 3.1 交易方向筛选

**Selling（出售）：**
- 筛选Have中包含当前道具的订单
- 用户视角：我想买这个道具

**Buying（求购）：**
- 筛选Want中包含当前道具的订单
- 用户视角：我想卖这个道具

### 3.2 NLF过滤逻辑

**排除NLF筛选开关：**
- 开启时：隐藏NLF中包含用户拥有道具的订单
- 需要用户登录才能使用
- 基于用户的道具库存信息

### 3.3 快速报价

**报价弹窗：**
- 简化版报价表单
- 选择报价道具
- 显示Value对比
- 提交报价

**报价结果：**
- 成功：提示并可跳转订单详情
- 失败：显示错误原因

### 3.4 数据刷新

**自动刷新：**
- 新订单提示（不自动刷新，避免列表跳动）
- 用户可手动点击刷新

**实时状态：**
- 订单状态实时更新
- 已完成/取消的订单标记或隐藏

## 四、SEO优化

### 4.1 TDK设置

**Title格式：**
- [道具名称] Trades - Buy & Sell [道具名称] | [平台名称]

**Description格式：**
- Find the best [道具名称] trades. [X] active listings. Trade [道具名称] with trusted players.

**H1：**
- [道具名称] Trades

### 4.2 URL结构

**路由设计：**
- /items/[item-slug]/trades
- 或 /trades/item/[item-slug]

**URL参数：**
- direction: selling/buying
- sort: 排序方式
- page: 页码
- minValue/maxValue: Value范围

### 4.3 结构化数据

**Product结构化数据：**
```json
{
  "@type": "Product",
  "name": "道具名称",
  "description": "道具描述",
  "offers": {
    "@type": "AggregateOffer",
    "offerCount": "活跃交易数",
    "lowPrice": "最低Value",
    "highPrice": "最高Value"
  }
}
```

### 4.4 FAQ结构化数据

**常见问题：**
- How to trade [道具名称]?
- What is the value of [道具名称]?
- How to avoid scams when trading [道具名称]?
